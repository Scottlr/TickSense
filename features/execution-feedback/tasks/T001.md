### T001 - Feedback Domain Model

#### Objective
Add a Java 11-safe domain model for after-the-fact execution feedback. This creates the shared vocabulary before any strategy, tracker, UI, or sound code emits feedback.

#### Scope
- Create feedback types under `src/main/java/com/ticksense/activities/feedback/`.
- Add immutable value objects for feedback events and evidence.
- Add a sink/buffer pattern similar to activity diagnostics and opportunity markers.
- Add unit tests for validation, immutability, and feedback type semantics.

#### Out of Scope
- Do not render feedback.
- Do not play sound.
- Do not infer misclicks.
- Do not emit feedback from existing strategies yet.
- Do not change report JSON in this task.

#### Blocked By
None

#### Why It Is Blocked
This is foundational and can start immediately because it only adds new domain types.

#### Related Code
- `src/main/java/com/ticksense/activities/ActivityDiagnostic.java` - simple immutable activity evidence model.
- `src/main/java/com/ticksense/activities/OpportunityMarker.java` - immutable marker shape with `EventTime`, context, and evidence.
- `src/main/java/com/ticksense/activities/ActivityDiagnosticBuffer.java` - bounded in-memory collector pattern.
- `src/main/java/com/ticksense/common/ImmutableCollections.java` - local immutable copy helpers.
- `src/test/java/com/ticksense/activities/ActivityStrategyEngineTest.java` - examples for activity lifecycle assertions.

#### Implementation Details
Create `src/main/java/com/ticksense/activities/feedback/ExecutionFeedbackType.java`:

```java
public enum ExecutionFeedbackType
{
    PERFECT_TICK,
    LIKELY_MISCLICK,
    DOUBLE_INPUT,
    LATE,
    MISSED,
    RECOVERED
}
```

Create `ExecutionFeedbackEvidence`:

```java
public final class ExecutionFeedbackEvidence
{
    private final EventTime time;
    private final String sourceType;
    private final EvidenceStrength strength;
    private final String detail;
}
```

Create `ExecutionFeedbackEvent`:

```java
public final class ExecutionFeedbackEvent
{
    private final String feedbackId;
    private final ActivityId activityId;
    private final ExecutionFeedbackType type;
    private final String label;
    private final EventTime time;
    private final double confidence;
    private final Map<String, String> context;
    private final List<ExecutionFeedbackEvidence> evidence;
}
```

Create `ExecutionFeedbackSink`:

```java
@FunctionalInterface
public interface ExecutionFeedbackSink
{
    void accept(ExecutionFeedbackEvent feedback);
}
```

Create `ExecutionFeedbackBuffer` as a bounded collector with constructor `ExecutionFeedbackBuffer(int maxEvents)`, `accept(ExecutionFeedbackEvent)`, and `snapshot()`.

#### Migration Notes
No existing APIs should break. This task only adds new classes.

#### Misinterpretations To Prevent
- Wrong interpretation: feedback events are UI commands.
  Correct rule: feedback events are domain facts; UI decides presentation later.
- Wrong interpretation: `PERFECT_TICK` can mean "good enough".
  Correct rule: `PERFECT_TICK` means confirmed zero tick loss only.
- Wrong interpretation: likely misclick is certain.
  Correct rule: label and type must preserve uncertainty.

#### Invariants
- Every feedback event has a non-empty label.
- Confidence is between `0.0D` and `1.0D`.
- Evidence and context are immutable copies.
- No RuneLite imports appear in `com.ticksense.activities.feedback`.
