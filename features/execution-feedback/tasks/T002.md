### T002 - Click Position Telemetry

#### Objective
Capture sanitized canvas click coordinates in normalized player-action telemetry so later tasks can infer likely near-target misclicks after a correction is observed.

#### Scope
- Add optional canvas click coordinates to `PlayerActionTelemetryEvent`.
- Capture `Client.getMouseCanvasPosition()` in the RuneLite adapter path for `MenuOptionClicked`.
- Serialize and replay the new fields with backward-compatible defaults.
- Add adapter and JSON tests.

#### Out of Scope
- Do not classify misclicks.
- Do not render click positions.
- Do not store raw mouse movement history.
- Do not capture coordinates for every client tick.

#### Blocked By
T001

#### Why It Is Blocked
The coordinate fields exist to support execution feedback evidence. The domain model should exist before adding new evidence sources.

#### Implementation Details
Modify `PlayerActionTelemetryEvent`:

```java
private final int canvasX;
private final int canvasY;

public int getCanvasX()
public int getCanvasY()
public boolean hasCanvasPoint()
```

Keep a compatibility constructor or update all call sites to pass `-1, -1`. Prefer a full constructor:

```java
public PlayerActionTelemetryEvent(
    EventTime time,
    Map<String, String> tags,
    String option,
    String target,
    EntityRef targetRef,
    String actionKind,
    WorldLocation location,
    int menuActionId,
    int canvasX,
    int canvasY)
```

Update `TelemetryJson` payload:

```java
payload.put("canvasX", e.getCanvasX());
payload.put("canvasY", e.getCanvasY());
```

For backwards compatibility in `fromJsonLine`, missing fields should default to `-1`.

Update `RuneLiteEventAdapter.mapMenuOptionClicked` to accept the click point. If direct adapter access to `Client` would break separation, add the point to the `RuneLiteEventEnvelope` or pass a small immutable value from `TickSensePlugin.onMenuOptionClicked`.

Use RuneLite source: `Client.getMouseCanvasPosition()` returns `net.runelite.api.Point`. Store only integer x/y, not a RuneLite `Point`.

#### Migration Notes
Existing JSON fixtures without `canvasX` or `canvasY` must still replay. Use missing-field defaults in `TelemetryJson`.

#### Misinterpretations To Prevent
- Wrong interpretation: capture all mouse movement.
  Correct rule: capture only the click point associated with player action events.
- Wrong interpretation: use RuneLite `Point` in core telemetry.
  Correct rule: normalized telemetry stores primitive `int` fields only.
- Wrong interpretation: coordinates alone prove a misclick.
  Correct rule: coordinates are only one evidence input for T005.

#### Invariants
- Unknown click point is `-1, -1`.
- Core telemetry does not import `net.runelite.api.Point`.
- Old replay fixtures still pass.
- Coordinate capture must not store account names, chat, or screenshots.

## Final Notes

- Recommended implementation order: after T001, before T005.

- Risks: `getMouseCanvasPosition()` may reflect current mouse position after the event rather than exact press position; verify with debug capture before tuning thresholds.
