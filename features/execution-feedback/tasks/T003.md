### T003 - Feedback Config Surface

#### Objective
Add player-facing configuration for immediate after-the-fact feedback, with separate controls for positive Perfect Tick feedback and correction/mistake feedback.

#### Scope
- Add config items to `TickSenseConfig`.
- Add small enums for placement and sensitivity.
- Include config values in debug export snapshots.
- Add tests for defaults through existing plugin/service config test patterns.

#### Out of Scope
- Do not render text.
- Do not play sound.
- Do not emit feedback.
- Do not add overlay classes yet.

#### Blocked By
T001

#### Why It Is Blocked
Config names and toggles should map to the feedback domain types from T001.

#### Related Code
- `src/main/java/com/ticksense/runelite/TickSenseConfig.java` - config interface.
- `src/main/java/com/ticksense/runelite/TickSensePlugin.java` - `configSnapshotProvider`.
- `src/main/java/com/ticksense/ui/TickSensePanel.java` - current panel receives config.
- `src/test/java/com/ticksense/runelite/TickSensePluginTest.java` - plugin boot/config patterns.

#### Implementation Touchpoints
- Read first: `TickSenseConfig`, `TickSensePlugin.configSnapshotProvider`.
- Edit first: `TickSenseConfig`.
- Symbols to search: `debugEventRecorder`, `debugActivityDiagnostics`, `configSnapshotProvider`.
- Required structural changes: add `PerfectTickFeedbackPlacement` and `MisclickSensitivity` enums.
- Do not edit: execution trackers, telemetry events, overlays.
- Verify in order: `.\gradlew.bat test --tests com.ticksense.runelite.*`, then `.\gradlew.bat test`.
- Stop and ask: if default-on feedback is requested; default should be conservative.

#### Implementation Details
Create `src/main/java/com/ticksense/runelite/PerfectTickFeedbackPlacement.java`:

```java
public enum PerfectTickFeedbackPlacement
{
    ABOVE_PLAYER,
    XP_DROP_STYLE,
    PANEL_ONLY
}
```

Create `src/main/java/com/ticksense/runelite/MisclickSensitivity.java`:

```java
public enum MisclickSensitivity
{
    CONSERVATIVE,
    NORMAL
}
```

Add config methods to `TickSenseConfig`:

```java
default boolean executionFeedback()
default boolean perfectTickText()
default boolean perfectTickSound()
default boolean correctionFeedbackText()
default PerfectTickFeedbackPlacement feedbackPlacement()
default int feedbackCooldownTicks()
default MisclickSensitivity misclickSensitivity()
```

Recommended defaults:

- `executionFeedback()` returns `false` until UI/sound behavior is reviewed.
- `perfectTickText()` returns `true`.
- `perfectTickSound()` returns `false`.
- `correctionFeedbackText()` returns `false`.
- `feedbackPlacement()` returns `PerfectTickFeedbackPlacement.ABOVE_PLAYER`.
- `feedbackCooldownTicks()` returns `2`.
- `misclickSensitivity()` returns `MisclickSensitivity.CONSERVATIVE`.

Update `configSnapshotProvider` to include all new config values.

#### Migration Notes
Existing users get default config values from RuneLite. No storage migration is needed.

#### Misinterpretations To Prevent
- Wrong interpretation: one toggle controls everything.
  Correct rule: master toggle disables all feedback, but text and sound remain separately configurable.
- Wrong interpretation: mistake feedback should default on.
  Correct rule: correction/mistake text defaults off until reviewed.

#### Invariants
- Master feedback toggle must suppress visual and audio feedback.
- Perfect sound has its own toggle.
- Correction feedback has its own toggle.
- Config names must be player-facing and avoid implementation jargon.

#### Acceptance Criteria
- [ ] Config methods and enums exist.
- [ ] Debug export snapshot includes the new values.
- [ ] Tests or static assertions confirm default values.
- [ ] `.\gradlew.bat test` passes.

#### Testing Guidance
Add a focused test if existing config tests can instantiate a simple config implementation. Otherwise assert config snapshot keys through `TickSensePlugin` test utilities.

#### Agent Handoff Context
This task makes the feature opt-in and clearly separable. It does not make feedback visible yet.
