```java
@ConfigGroup("ticksense")
public interface TickSenseConfig extends Config
{
    @ConfigItem(keyName = "debugEventRecorder", name = "Debug event recorder", description = "Write local normalized debug timelines.")
    default boolean debugEventRecorder()
    {
        return false;
    }

    @ConfigItem(keyName = "debugActivityDiagnostics", name = "Activity diagnostics", description = "Show developer-only activity diagnostics.")
    default boolean debugActivityDiagnostics()
    {
        return false;
    }

    @ConfigItem(keyName = "maxDebugFileSizeMb", name = "Max debug file size MB", description = "Maximum debug file size.")
    default int maxDebugFileSizeMb()
    {
        return 25;
    }

    @ConfigItem(keyName = "maxDebugSessions", name = "Max debug sessions", description = "Maximum retained debug sessions.")
    default int maxDebugSessions()
    {
        return 5;
    }
}
```

#### Migration Notes

#### Misinterpretations To Prevent
- Wrong interpretation: the panel should show live activity status by default.
  Correct rule: normal UI is report-only; live diagnostics appear only when debug config is enabled in later tasks.
- Wrong interpretation: debug config means debug files are written now.
  Correct rule: T002 only defines config keys; T007 implements debug recording.

#### Invariants
- Config defaults keep debug disabled.
- Panel contains no real-time advice, target suggestions, or mechanic prompts.
- UI package may depend on RuneLite UI/report models but not strategy internals.
