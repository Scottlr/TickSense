#### Migration Notes
- No existing APIs break.
- Constructor signatures should remain stable once telemetry/storage tasks depend on them.

#### Misinterpretations To Prevent
- Wrong interpretation: import RuneLite `WorldPoint` or `NPC` for convenience.
  Correct rule: use `WorldLocation` and `EntityRef` snapshots only.
- Wrong interpretation: make these mutable so strategies can fill them later.
  Correct rule: keep values immutable; create new values for state changes.

#### Invariants
- `com.ticksense.core` must compile without RuneLite, Swing, storage, or non-template runtime dependencies.
- `EventTime` always carries both game tick and local time fields, even when some values use sentinel defaults.
- `FinishReason` includes confidence and evidence.

#### Acceptance Criteria
- [ ] Core primitives compile under Java 11.

#### Tests Required
