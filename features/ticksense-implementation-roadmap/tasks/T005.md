#### Migration Notes
- Schema version starts at `1`.
- Add explicit unknown/default handling in `TelemetryJson` so future schema changes can fail clearly instead of silently corrupting replay.

#### Misinterpretations To Prevent
- Wrong interpretation: store all payload fields in `Map<String, String>`.
  Correct rule: use explicit fields for common data and tags only for extension/source metadata.
- Wrong interpretation: serialize RuneLite objects directly.
  Correct rule: telemetry JSON contains primitives, strings, lists, maps, and TickSense value objects only.

#### Invariants
- Every event has `schemaVersion`, event type, category, time, and tags.
- Telemetry payloads are immutable after construction.
- Serialization round trips preserve behaviorally relevant fields.
