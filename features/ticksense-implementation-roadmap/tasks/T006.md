#### Migration Notes
- Adapter-internal envelope fields may change, but telemetry JSON must stay schema-versioned.
- If a mapper cannot confidently produce telemetry, it should return `Optional.empty()` and optionally emit debug diagnostics in later tasks.

#### Misinterpretations To Prevent
- Wrong interpretation: strategies can inspect `NpcSpawned` directly.
  Correct rule: strategies consume only `TelemetryEvent`.
- Wrong interpretation: adapter mapping can decide that a boss or gem mining activity started.
  Correct rule: adapter maps evidence only; activity decisions belong to strategies.

#### Invariants
- `com.ticksense.core`, `com.ticksense.telemetry`, `com.ticksense.activities`, `com.ticksense.analytics`, and `com.ticksense.storage` do not import RuneLite APIs.
- Every mapped telemetry event includes timing from the event envelope.
- Mapper tests cover behavior, not framework wiring.
