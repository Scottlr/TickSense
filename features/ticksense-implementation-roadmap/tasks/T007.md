### T007 - Telemetry Bus, Session Clock, and Debug Recorder

#### Goal
Fan out normalized telemetry, maintain session/tick sequencing, and write bounded opt-in debug JSONL of normalized events.

#### Scope
- Add `TelemetrySink`, `TelemetryBus`, and `SessionTelemetryContext`.
- Wire `RuneLiteEventCapture` and `RuneLiteEventAdapter` to publish telemetry envelopes to the bus.
- Add `DebugEventRecorder` that writes normalized telemetry JSONL only when `debugEventRecorder=true`.
- Enforce max debug file size and retained session count from config.

#### Out of scope
- No timeline repository for product storage; T008 owns durable timelines.
- No activity diagnostics UI; T021 owns developer diagnostics panel/export.
- No raw RuneLite event serialization.

#### Implementation Notes
- Suggested interfaces:

```java
public interface TelemetrySink
{
    void accept(TelemetryEnvelope envelope);
}

public final class TelemetryBus implements TelemetrySink
{
    public void addSink(TelemetrySink sink);
    public void removeSink(TelemetrySink sink);
    @Override
    public void accept(TelemetryEnvelope envelope);
}
```

- `DebugEventRecorder` writes under `~/.runelite/ticksense/debug/`.
- On file write failure, log one warning with enough path/error context and disable that recorder instance.

#### Before / After
Before:

```java
adapter.mapMenuOptionClicked(event, envelope);
```

After:

```java
adapter.mapMenuOptionClicked(event, envelope)
    .ifPresent(telemetryBus::accept);
```

#### Migration Notes
- Debug JSONL uses the same schema as telemetry JSON to avoid a separate debug format.
- Debug files are developer artifacts and not the durable product timeline.

#### Misinterpretations To Prevent
- Wrong interpretation: debug recorder can store raw RuneLite events for convenience.
  Correct rule: debug recorder writes normalized telemetry envelopes only.
- Wrong interpretation: bus failures should crash the plugin.
  Correct rule: sink failures are contained and logged as lifecycle/storage diagnostics.

#### Invariants
- Debug recording is disabled by default.
- Debug capture is bounded by size and retained session count.
- No per-event production logs are emitted.

#### Acceptance Criteria
- [ ] Telemetry bus fans out events to multiple sinks in order.
- [ ] A failing sink does not prevent later sinks from receiving events.
- [ ] Debug recorder writes JSONL only when enabled.
- [ ] Debug retention respects config limits.
- [ ] `./gradlew test` passes.

#### Tests Required

#### Agent Handoff Context
This task gives later strategy and storage tasks a clean event stream. Keep debug useful but modest; full diagnostics/export are separate post-MVP work.
