### T008 - Opportunity Model and Lifecycle

#### Goal
Add the opportunity domain model and lifecycle service so activities can measure start, completion, failure, expiry, cancellation, evidence, and latency.

#### Scope
- Create `OpportunityDefinition`, `OpportunityInstance`, `OpportunityStatus`, `OpportunityEvidence`, `EvidenceStrength`, `OpportunityLifecycle`, `OpportunitySink`, and `OpportunityMarker`.
- Add latency helpers for milliseconds and game ticks.
- Add terminal-state behavior for completed, failed, expired, and cancelled opportunities.
- Allow strategies to define opportunity types without embedding metrics calculations.

#### Out of scope
- No gem mining, Araxxor, or other activity-specific opportunity definitions.
- No report scoring; T016 owns generic metrics and scoring.
- No UI timeline rendering; T018 owns display.

#### Implementation Notes
- Required status enum:

```java
public enum OpportunityStatus
{
    OPEN,
    COMPLETED,
    FAILED,
    EXPIRED,
    CANCELLED
}
```

- Required core class shape:

```java
public final class OpportunityInstance
{
    private final String instanceId;
    private final ActivityId activityId;
    private final OpportunityDefinition definition;
    private final EventTime startTime;
    private final Map<String, String> context;
    private final List<OpportunityEvidence> evidence;
    private EventTime endTime;
    private OpportunityStatus status;

    public long latencyMillis();
    public int latencyTicks();
}
```

- `OpportunityLifecycle` should reject terminal-state changes after an instance is no longer `OPEN`.
- `OpportunityMarker` fields: `String markerId`, `String opportunityInstanceId`, `ActivityId activityId`, `String opportunityType`, `OpportunityStatus status`, `EventTime time`, `Map<String, String> context`, `List<OpportunityEvidence> evidence`.

#### Before / After

After:

```java
OpportunityInstance op = opportunityLifecycle.start(activityId, definition, startTime, context);
opportunityLifecycle.complete(op.getInstanceId(), endTime, evidence);
```

#### Migration Notes
- T010 persists opportunity markers using this task's `OpportunityMarker`/`OpportunityInstance` fields.
- Keep field names stable because T017 report models and T019 golden fixtures will depend on them.

#### Misinterpretations To Prevent
- Wrong interpretation: opportunities are just UI timeline rows.
  Correct rule: opportunity instances are the analytics source of truth.
- Wrong interpretation: failed and expired are equivalent.
  Correct rule: `FAILED` means failure evidence occurred; `EXPIRED` means no response before timeout.

#### Invariants
- Latency returns a sentinel only while the opportunity is open; terminal states have end time.
- Terminal states cannot be overwritten.
- Evidence includes strength and source event type.

#### Acceptance Criteria
- [ ] `OpportunityLifecycle` can start, complete, fail, expire, and cancel opportunities.
- [ ] `OpportunityLifecycle` computes latency ticks and millis from `EventTime`.
- [ ] `OpportunityLifecycle` preserves evidence in insertion order.
- [ ] `OpportunityLifecycle` emits opportunity markers that T010 can persist and T019/T020 can replay into reports.
- [ ] `./gradlew test` passes.
