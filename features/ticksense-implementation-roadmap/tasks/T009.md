#### Implementation Notes
- Required interface:

```java
public interface ActivityStrategy
{
    ActivityDefinition getDefinition();
    ActivityCandidate evaluateActivation(ActivityContext context, TelemetryEvent event);
    void onStart(ActivityContext context, ActivitySession session);
    void onEvent(ActivityContext context, ActivitySession session, TelemetryEvent event, OpportunitySink sink);
    Optional<FinishReason> evaluateTermination(ActivityContext context, ActivitySession session, TelemetryEvent event);
    ActivityReportData buildActivityData(ActivityContext context, ActivitySession session);
}
```

- `ActivityDefinition` fields: `ActivityType activityType`, `String displayName`, `int priority`, `double activationThreshold`, `boolean bossActivity`.
- `ActivityCandidate` fields should include `activityType`, `confidence`, `evidenceSummary`, `suppressed`, and `suppressionReason`.
- `ActivityRegistry` should expose immutable registered strategies and preserve deterministic ordering.
- `ActivityRegistry` and `ActivityStrategyFactory` must not expose per-session manual selection APIs. A later config may enable or disable entire activity families, but active sessions are always selected by strategy confidence.
- `ActivityMarker` fields: `String markerId`, `ActivityId activityId`, `ActivityType activityType`, `String markerType`, `EventTime time`, `Map<String, String> metadata`.

#### Migration Notes
- Strategy contracts become central APIs. Keep them minimal and stable.
- If a method needs more context later, prefer adding fields to `ActivityContext` over passing RuneLite objects.

#### Misinterpretations To Prevent
- Wrong interpretation: use inheritance hierarchy per boss/skilling family before actual duplication exists.
  Correct rule: start with one small strategy interface and concrete strategies.
- Wrong interpretation: registry should discover classes dynamically.
  Correct rule: use explicit source-owned factory/registry; analyzer logic is compiled into the plugin.
- Wrong interpretation: contracts should support a normal panel control for selecting or switching the current activity.
  Correct rule: strategies declare confidence from telemetry; normal users never manually select, start, stop, or switch an activity.

#### Invariants
- Strategy contracts are pure Java.
- Activities own lifecycle decisions.
- Activity activation is automatic and confidence/evidence based.
- Registry order is deterministic for repeatable tests.
