### T014 - Araxxor ID Registry and Verification Gates

#### Goal
Create source-owned Araxxor ID registries and code-level gates that prevent unverified Araxxor mechanics from entering normal reports.

#### Scope
- Create `AraxxorIds` with explicit sets for boss NPC IDs, spider NPC IDs, relevant graphics/projectiles if verified, and optional region IDs.
- Add comments for each committed ID source: RuneLite constant name as text, existing source URL/path, or Dev Tools verification date.
- Add tests that fail if required ID sets are empty when the strategy is enabled.
- Add `AraxxorVerificationStatus` and disable normal Araxxor strategy registration unless status is `VERIFIED`.

#### Out of scope
- No Araxxor strategy logic; T015 owns detection and opportunities.
- No live overlay, live target highlighting, or real-time mechanic guidance.
- No arbitrary user-defined ID packs.
- No captured replay fixtures; T029 owns verification capture fixtures and blocked/verified decisions.
