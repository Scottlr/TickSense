### T019 - Replay and Golden Timeline Harness

#### Goal
Add replay and golden report tests that run activity strategies and analytics without RuneLite.

#### Scope
- Create `TimelineReplayRunner`.
- Create `TestEvents` synthetic telemetry builder.
- Add JSONL replay fixtures for the MVP auto-detected local activity: gem mining.
- Add an ambiguous/low-confidence replay fixture that must not produce a normal report.
- Add golden report JSON comparison helpers.
- Cover known edge cases: inventory update delayed by a tick, duplicate clicks, gem mining RNG separation, logout during opportunity, and ambiguous activity evidence.

#### Out of scope
- No new production strategy behavior except fixes needed to make replay behavior correct.
- No full UI automation.
- No tests for trivial data classes.

#### Migration Notes
- Golden fixtures are part of the report schema contract.
- If a report format changes intentionally, update golden JSON and note why in the PR.

#### Misinterpretations To Prevent
- Wrong interpretation: replay tests need RuneLite running.
  Correct rule: replay consumes normalized telemetry and pure Java core only.
- Wrong interpretation: golden tests can ignore opportunity timelines to avoid churn.
  Correct rule: golden reports must include metrics and key opportunities.
- Wrong interpretation: ambiguous replays should prompt for user selection to make tests deterministic.
  Correct rule: ambiguous replays assert that no normal report is created.

#### Invariants
- Replay tests do not require RuneLite.
- Fixtures contain anonymized normalized telemetry only.
- Regression tests protect behavior named in the architecture.
- No replay fixture depends on post-release combat work for MVP.
