### T020 - MVP End-to-End Assembly and Plugin Hub Readiness

#### Goal
Compose the MVP pipeline so a completed activity can be captured, normalized, stored, analyzed after completion, saved as a report, and displayed in the RuneLite panel.

#### Scope
- Compose `RuneLiteEventCapture`, `RuneLiteEventAdapter`, `TelemetryBus`, `JsonlTimelineRepository`, `ActivityStrategyEngine`, `OpportunityTracker`, analyzers, `JsonReportRepository`, and `TickSensePanel`.
- Register the MVP strategy: gem mining.
- Ensure analysis runs after finish markers, not during live activity.
- Add Plugin Hub readiness checks for Java 11, dependencies, data paths, and compliance boundaries.

#### Out of scope
- No new activities.
- No non-local runtime components, non-JSONL database runtime, live overlays, sounds, input generation, menu mutation, or arbitrary rule packs.
- No normal-user manual activity selection, start, stop, or switch controls.
- No broad redesign of already implemented packages.

#### Implementation Notes
- Suggested composition root:

```java
public final class TickSenseServices implements AutoCloseable
{
    private final TelemetryBus telemetryBus;
    private final TimelineRepository timelineRepository;
    private final ReportRepository reportRepository;
    private final ActivityStrategyEngine strategyEngine;
    private final ReportGenerationService reportGenerationService;

    public void start();
    @Override
    public void close();
}
```

- `ReportGenerationService` subscribes to finished activity markers, reads the activity timeline, runs analytics, saves the report, and notifies the panel.
- Normal UI update happens after report save.

#### Misinterpretations To Prevent
- Wrong interpretation: analyze and show opportunity status during live activity.
  Correct rule: analysis and normal UI display happen after finish reason.
- Wrong interpretation: add external summaries or trends because reports exist.
  Correct rule: MVP ends with local retrospective reports only.
- Wrong interpretation: add a manual activity picker because the MVP has one concrete activity.
  Correct rule: gem mining must be automatically detected; ambiguous/no-confidence timelines produce no normal report.

#### Invariants
- End-to-end MVP remains observe-only and retrospective.
- All core analytics can run without RuneLite.
- Plugin shuts down cleanly and closes repositories.
- MVP registration contains gem mining only; boss strategies are post-MVP.
