### T021 - Optional Developer Diagnostics and Debug Export Bundle

#### Goal
Add optional post-MVP developer diagnostics and user-requested debug export bundles for completed reports and captured normalized timelines.

#### Scope
- Add debug-only `DeveloperDiagnosticsPanel`.
- Show active strategy, candidate strategies/confidence, current region/world-view metadata, last 50 normalized events, open opportunities, last finish reason, and unknown IDs when debug is enabled.
- Add `ExportBundleWriter` that creates a local zip with `bundle.json`, `plugin-config.json`, `report.json`, `activity.json`, `timeline.jsonl`, and `diagnostics.jsonl`.
- Keep export user-requested and local.
- Keep all diagnostics hidden from normal MVP UX unless a developer/debug toggle is enabled.

#### Out of scope
- No production per-event logs.
- No MVP requirement; this task must remain optional post-MVP tooling.
- Export writes a local bundle only; no upload, sync, or server component.
- No diagnostics beyond the debug recorder/dev-only diagnostics described in the architecture.

#### Migration Notes

#### Misinterpretations To Prevent
- Wrong interpretation: diagnostics justify a live overlay for current mechanics.
  Correct rule: diagnostics are developer-only and hidden unless debug is enabled.
- Wrong interpretation: export can upload directly to support.
  Correct rule: export writes a local user-owned bundle only.

#### Invariants
- Diagnostics are opt-in/debug-only.
- Diagnostics/export are optional post-MVP tooling and do not block MVP readiness.
- Bundle contains normalized/debug/report data, not raw RuneLite objects.
- Export writes local files only.
