### T022 - Retention, Trends, and Report Index Maintenance

#### Goal
Keep local report storage manageable and show simple local trends across recent completed reports.

#### Scope
- Add retention settings for raw timelines and report summaries.
- Add report index rebuild and compaction commands.
- Add `TrendAnalyzer` for local report summaries such as median latency change and repeated tick-loss categories.
- Add a basic Trends tab fed by stored report summaries.

#### Out of scope
- No SQL migration in this task.
- No account identity or non-local trend source.

#### Migration Notes
- Retention deletes only files under TickSense data root.
- Index maintenance must be compatible with schema version `1` reports.

#### Misinterpretations To Prevent
- Wrong interpretation: trends require raw telemetry uploads.
  Correct rule: trends are local and report-summary based.
- Wrong interpretation: a database should be added now because trends need queries.
  Correct rule: JSON/index maintenance remains the post-MVP next step; any runtime dependency requires owner approval.

#### Invariants
- Local report summaries only.
- Trend metrics compare like activity/report schemas.
- Retention never deletes outside TickSense data root.
