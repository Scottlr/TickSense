### T030 - Gem Mining Strategy and Opportunities

#### Goal
Detect gem mining activity and execution opportunities using verified normalized telemetry.

#### Scope
- Create `GemMiningStrategy` and `GemMiningState`.
- Emit `GEM_ROCK_RESPAWN_TO_CLICK`, `GEM_MINING_IDLE`, and `GEM_MOVEMENT_TO_ROCK` opportunities.
- Automatically detect activity start, rock click, mining animation/XP/inventory confirmation, idle ticks, redundant clicks, and termination.

#### Out of scope
- No analyzer/report formatting; T031 owns metrics and golden reports.
- No live rock highlights or click suggestions.
- No unverified object IDs.
- No normal-user manual activity selection or start/stop controls.

#### Implementation Notes
- Redundant clicks require repeated same rock action without animation, XP, inventory, or target progress.
- Mining RNG must not be converted into execution loss.
- Activation confidence must come from verified rock/player evidence; low-confidence activity stays suppressed by T011.

#### Misinterpretations To Prevent
- Wrong interpretation: report every failed ore roll as idle time.
  Correct rule: idle ticks only accrue when progress was possible and no relevant action occurred.
- Wrong interpretation: users should select gem mining before they start.
  Correct rule: the strategy detects gem mining from normalized telemetry and confidence evidence.

#### Invariants
- Consumes normalized telemetry only.
- Auto-detected activity; no manual session controls.
- No live helper behavior.
- Execution loss is separated from mining RNG.

#### Acceptance Criteria
- [ ] Strategy starts from verified gem rock evidence.
- [ ] Strategy remains inactive when evidence does not reach activation confidence.
- [ ] Strategy records respawn-to-click latency.
- [ ] Strategy records idle ticks without penalizing RNG.
- [ ] `./gradlew test` passes.
